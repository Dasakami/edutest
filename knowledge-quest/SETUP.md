# Настройка системы тестирования

## Требования

- Node.js 16+ и npm
- FastAPI бэкенд, запущенный на `http://localhost:8000`

## Установка

1. Установите зависимости:
```bash
npm install
```

2. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

3. Настройте переменную окружения в `.env`:
```
VITE_API_BASE_URL=http://localhost:8000/api
```

## Запуск приложения

```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:8080`

## Функциональность

### Для студентов
- ✅ Регистрация и вход
- ✅ Просмотр доступных тестов
- ✅ Прохождение тестов с таймером
- ✅ Просмотр результатов

### Для преподавателей
- ✅ Регистрация и вход
- ✅ Создание тестов
- ✅ Добавление вопросов (один ответ, несколько ответов, текстовый)
- ✅ Редактирование тестов
- ✅ Просмотр статистики

## Использование

1. Зарегистрируйтесь как студент или преподаватель
2. Войдите в систему
3. **Студент**: выберите тест из списка и пройдите его
4. **Преподаватель**: создайте тест, добавьте вопросы и просматривайте статистику

## API Endpoints

Приложение использует следующие эндпоинты:
- `POST /api/auth/register` - регистрация
- `POST /api/auth/login` - вход
- `GET /api/users/me` - текущий пользователь
- `GET /api/tests` - список тестов
- `POST /api/tests` - создание теста
- `GET /api/tests/{id}` - получение теста
- `POST /api/questions` - создание вопроса
- `POST /api/results/submit` - отправка результатов
- `GET /api/results/my` - мои результаты
- `GET /api/results/test/{id}` - результаты теста
- `GET /api/results/statistics/{id}` - статистика теста

## Дизайн система

Приложение использует:
- **Основной цвет**: Синий (образование, профессионализм)
- **Акцентный цвет**: Зеленый (успех)
- **Деструктивный**: Красный (ошибки, неудачи)
- **Tailwind CSS** для стилизации
- **Shadcn UI** для компонентов
- **HSL цвета** для гибкой темизации

## Токены авторизации

Приложение автоматически:
- Сохраняет JWT токен в localStorage
- Добавляет токен к каждому запросу
- Обновляет токен при необходимости
- Перенаправляет на страницу входа при 401 ошибке
